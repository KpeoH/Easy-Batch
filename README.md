# EasyBatch

Десктопное приложение для многопоточной пакетной обработки изображений. Инструмент предназначен для автоматизации рутинных задач по подготовке визуального контента: интеллектуальный ресайз, наложение водяных знаков, кроп и генерация обложек с метаданными.

## Основной функционал

* **Пакетная обработка:** Асинхронный экспорт файлов с использованием `concurrent.futures.ThreadPoolExecutor`. Обеспечивает утилизацию всех ядер CPU и предотвращает блокировку основного потока UI при работе с большими объемами данных.
* **Система водяных знаков:**
  * Поддержка текстовых и растровых (PNG) вотермарок.
  * Тонкая настройка прозрачности (альфа-канал), масштаба, позиционирования (9 якорей) и смещения по осям X/Y.
  * Режим защитной сетки: автоматическое замощение холста водяным знаком с поворотом на 30 градусов и расчетом шага сетки.
* **Генерация обложек:**
  * Автоматическое наложение дополнительных слоев: рекламных плашек и логотипов уценки.
  * Программный рендеринг ценников (зачеркнутая старая цена и новая цена) с вычислением bounding box для корректного отображения шрифтов (поддержка кастомных TTF).
* **Кроп и ресайз:** Подгонка изображений под заданные разрешения с сохранением исходных пропорций (ImageOps.pad / ImageOps.fit). Настраиваемые шаблоны обрезки.
* **Интерфейс (GUI):** Написан на PySide6. Поддерживает Drag-and-Drop, динамическую сортировку файлов в памяти без перерисовки всего списка (нулевой Layout Reflow), предпросмотр в реальном времени. Встроены темная и светлая темы.
* **Сессии:** Кэширование пользовательских настроек, пресетов водяных знаков и координат кропа в локальные JSON-конфиги.

## Стек технологий

* Python 3.10+
* PySide6
* Pillow

## Установка и запуск (Среда разработки)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/KpeoH/Easy-Batch.git
   cd EasyBatch
   ```
2. Установите необходимые зависимости:
```Bash
pip install PySide6 Pillow
```

3. Запустите скрипт:
```Bash
python resizer.py
```


## Дополнительная информация:
Провёл нагрузочное стресс-тестирование десктопного пайплайна на батче из 10 000 изображений в разрешении 4K UHD (общий объём вывода — 20 000 сгенерированных файлов с обложками и водяными знаками).

Благодаря потоковой обработке через ThreadPoolExecutor (15 concurrent threads) и обходу GIL на уровне C-библиотеки Pillow, приложение продемонстрировало стабильную пропускную способность 8.37 фото/сек, выполнив весь объём за 19 минут.

Пиковое потребление ОЗУ при максимальной нагрузке выросло всего на +1.6 GB, что подтвердило отсутствие утечек памяти (Memory Leaks) при интенсивной работе Garbage Collector. Главный поток интерфейса (UI Thread) при этом оставался полностью отзывчивым
